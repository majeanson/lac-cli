2 featureKey format: domain-YYYY-NNN

Rationale

Year in key makes keys self-documenting. Domain prefix enables multi-domain workspaces (proc-, goal-, adr-).

Alternatives considered
  • UUIDs — opaque, not human-readable
  • sequential integers — no temporal context

lac export-all: batch all export formats to a folder

feat-2026-033
cli-export frozen 3 decisions

1 Registry array drives the loop rather than hardcoded if-chains

Rationale

Adding a new export format in the future requires one entry in the HTML_FORMATS array, not a new if-block. Mirrors the generator library's own open/closed structure.

Alternatives considered
  • Individual if-blocks per format (copied from export.ts)
  • Dynamic import via format name string

2 Scan features once, pass to all generators

Rationale

Avoids repeated filesystem walks. All generators accept the same ScannedFeature[] type, so one scan suffices for all 14+ formats.

Alternatives considered
  • Re-scan per format (simpler but slow for large workspaces)

3 Exclude postcard, markdown, and diff from export-all

Rationale

postcard and markdown are single-feature formats that require a nearest feature.json context. diff requires a second workspace path. None are meaningful in a batch context.

Alternatives considered
  • Include postcard using first frozen feature
  • Add --diff-target option

1 Two modes (direct + LSP) rather than LSP-only
